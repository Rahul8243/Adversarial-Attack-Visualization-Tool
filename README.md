## 📌 Project Title

**CIFAR-10 Adversarial Robustness Explorer** — a production-ready deep learning project that trains a CNN on CIFAR-10 and evaluates its vulnerability to three white-box adversarial attacks (FGSM, BIM, PGD), complete with an interactive Streamlit demo.

---

## 📖 Problem Statement

Modern deep neural networks achieve impressive accuracy on image classification benchmarks, yet they remain surprisingly fragile: adding imperceptibly small, carefully crafted noise to an image can flip a confident prediction entirely.

This project asks:
- *How much does accuracy drop under adversarial perturbations?*
- *Which attack is most effective at a given perturbation budget ε?*
- *Can we visualise and understand the perturbations intuitively?*

---

## 🎯 Objective

1. Train a compact CNN on CIFAR-10 (10-class image classification).
2. Implement FGSM, BIM, and PGD adversarial attacks from scratch in TensorFlow.
3. Benchmark clean vs. attacked accuracy across multiple ε values.
4. Provide a live, interactive UI for exploring attacks on arbitrary images.

## 🧠 Model / Approach Explanation

### Architecture — `cifar10_cnn`

| Layer            | Output Shape   | Params    |
|------------------|---------------|-----------|
| Conv2D (32, 3×3) | 32 × 32 × 32  | 896       |
| BatchNorm + Pool | 16 × 16 × 32  | 128       |
| Conv2D (64, 5×5) | 16 × 16 × 64  | 51 264    |
| BatchNorm + Pool | 8 × 8 × 64    | 256       |
| Conv2D (128, 7×7)| 8 × 8 × 128   | 401 536   |
| BatchNorm        | 8 × 8 × 128   | 512       |
| Flatten → Dense 128 + Dropout | —  | 1 049 600 |
| Dense (10, softmax) | 10         | 1 290     |

> Batch Normalisation and Dropout (0.4) are added over the original notebook for better generalisation.

### Adversarial Attacks

| Attack | Type | Key idea |
|--------|------|----------|
| **FGSM** | Single-step | Move each pixel by ε in the sign of the gradient |
| **BIM**  | Iterative   | Repeat FGSM with step α, project back to ε-ball |
| **PGD**  | Iterative + random start | BIM with random initialisation — hardest to defend against |

**Typical results** (ε = 0.03, 500 test samples):

| Attack   | ε    | Accuracy |
|----------|------|----------|
| Clean    | —    | ~75%     |
| FGSM     | 0.03 | ~35%     |
| BIM      | 0.03 | ~18%     |
| PGD      | 0.03 | ~12%     |

## 🚀 How to Run

### 1 — Clone & install

```bash
git clone https://github.com/Rahul8243/cifar10-adversarial.git
cd cifar10-adversarial
pip install -r requirements.txt
```

### 2 — Train the model

```bash
python train.py                    # default: 30 epochs, batch 64
python train.py --epochs 50        # more training
python train.py --epochs 5 --batch 128   # quick debug run
```

The trained model is saved to `model/cifar10_cnn.keras`.

### 3 — Run adversarial attacks (CLI)

```bash
# All attacks with default ε=0.03
python predict.py

# Specific attack
python predict.py --attack fgsm --epsilon 0.05

# PGD with custom iterations
python predict.py --attack pgd --epsilon 0.03 --iterations 40 --samples 200
```

### 4 — (Optional) Evaluate from Python

```python
from predict import load_model, pgd_attack, predict_single
from utils   import load_and_preprocess_data, get_sample_batch

model              = load_model()
_, _, x_test, y_test = load_and_preprocess_data()
x_s, y_s           = get_sample_batch(x_test, y_test, n=100)

adv   = pgd_attack(model, x_s, y_s, epsilon=0.03)
preds = predict_single(model, adv[0], top_k=5)
print(preds)
```

## 💻 Streamlit Usage

```bash
streamlit run app.py
```

Then open **http://localhost:8501** in your browser.

**UI Features:**
- 📂 Upload any image (auto-resized to 32×32)
- 🎲 Use a random CIFAR-10 test sample
- 🎚️ Select attack type (FGSM / BIM / PGD) and ε via sliders
- 📊 Top-5 confidence bars for original vs. adversarial
- 🔬 Perturbation heatmap with ℓ∞ and ℓ₂ distance metrics
- ✅ / ⚠️ Instant feedback on whether the attack succeeded

## 📊 Results

The model reaches ~74–78% clean test accuracy (baseline for CIFAR-10 with a 3-block CNN).  
Under PGD with ε = 0.03, accuracy drops to ~10–15%, demonstrating the stark vulnerability of undefended models.

![Attack Comparison](assets/attack_comparison.png)

*Results will vary slightly due to random initialisation in PGD.*

## 🔮 Future Improvements

- **Adversarial Training** — retrain with adversarial examples mixed into the training set (Madry et al., 2018).
- **Defensive Distillation** — use a teacher→student distillation pipeline to smooth decision boundaries.
- **AutoAttack** — evaluate with the parameter-free AutoAttack benchmark for fairer comparison.
- **ResNet / EfficientNet backbone** — replace the simple CNN with a stronger architecture for higher clean accuracy.
- **Targeted attacks** — modify FGSM/PGD to fool the model into predicting a *specific* wrong class.
- **Certifiable robustness** — integrate randomised smoothing for provable ℓ₂ robustness certificates.
- **Docker deployment** — containerise the Streamlit app for one-command cloud deployment.
